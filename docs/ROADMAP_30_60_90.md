# DeFAI Pal 30/60/90 Roadmap

## 30 Days (Stabilize and polish)

- Complete launch-token flow hardening:
  - token factory deployment verification
  - better in-chat validation for tax/burn parameters
  - auto-track newly created tokens in portfolio
- Improve transaction reliability:
  - stronger timeout handling
  - clearer recovery and retry messages
- Submission-level UX polish:
  - final spacing/contrast pass
  - improved empty states and helper copy

Success criteria:
- >95% success rate for demo transactions on supported flows
- End-to-end walkthrough with no blocking UX bugs

## 60 Days (Expand capabilities)

- Broaden protocol integrations:
  - richer lending adapters
  - additional swap route support
- Upgrade educational copilot quality:
  - topic packs (crypto basics, risk, portfolio construction)
  - context-aware explanations by user intent and experience level
- Analytics and user confidence:
  - action-level telemetry dashboard
  - conversion tracking from prompt to successful tx

Success criteria:
- More executable intents with consistent confirmation UX
- Improved prompt-to-success conversion rate

## 90 Days (Production readiness)

- Security and governance track:
  - independent audit preparation package
  - threat model and controls document
  - expanded test coverage for critical contracts
- Reliability upgrades:
  - failover RPC strategy
  - indexer-backed portfolio and activity views
- Ecosystem growth:
  - protocol partnerships on Polkadot ecosystem
  - community onboarding and feedback loop

Success criteria:
- Audit-ready smart contract package
- Roadmap-backed release cadence and public changelog

## Long-term commitment statement

DeFAI Pal is built as an ongoing product, not a one-off hackathon prototype. The team is committed to:

- iterating with real user feedback,
- strengthening security posture before production scale,
- expanding protocol integrations on Polkadot,
- and maintaining transparent roadmap and release communication.
